Interactive sin()function

A program which depends on user interaction to work

This is the initial screen of an interactive pattern. My code contains a sin() function which displays circles when the mouse enters the canvas. Depending on the mouse position, the size of the circle and the colour of the circle changes. For more information of this, see below.

Interactive sin()function

User interaction

To the right you can see an example of user interaction with my written sin()function. Here, the mouse causes the circles to be of a small size along with a green colour. The circle size is determined by the MouseY position, while the MouseX position determines the colour. In the first third of the screen (horizontally), there is a green colour, a red colour in the second third - and in the last third the colour randomizes.

Bubble popper game

Final assignment

For my final assignment, I used arrays, loops, functions and more. I decided to make a game based on the game Brick Breaker which was available on the original Blackberry. In order to have different screens, I used GAMESTATE to define each screen. I enjoyed figuring out how to make everything work. For more information and another visual regarding the game I made see below.

Bubble popper game

How to play the game

The game can be played with the a and d keys (to move left and right, respectively) with the black semi-circle. The red cicle bounces around the screen and can be used to get rid of the light blue "bubbles". The bubbles are part of an array, meaning that they can easily be multiplied if the developer wishes to do so.
